# Часть 1: Тест-кейсы оплаты банковской картой

**Условие задания**  
Веб-приложение позволяет оплатить заказ банковской картой через внешний платежный шлюз.  
Процесс:  
1. Клиентская валидация  
2. Отправка данных в шлюз  
3. Возможный редирект на 3-D Secure  
4. Шлюз возвращает результат (успех / отклонено банком / ошибка сервиса)  
5. Обновление статуса заказа  
6. Отображение результата пользователю  
7. Отправка чека на email (при успехе)

**Цель тест-кейсов**  
Полное покрытие:  
- Позитивный сценарий  
- Негативные сценарии  
- Граничные значения  
- Проверка клиентской валидации  
- Проверка интеграции с платежным шлюзом
